# 마로(Maro) @ui-brand UI/UX 디자인 원칙 가이드

> 마로(maro.ai.kr)의 AI 에이전트가 모든 UI 판단에서 즉시 참조할 수 있도록 구성한 학술 논문 근거 + 구체적 수치 기반 디자인 원칙 문서.
> 테라코타 `#c4756e` 브랜드 컬러, Gowun Batang 서체, 6단계 선물 추천 위저드, 모바일 우선 아키텍처 전제.

## 1. 색채 시스템: 테라코타의 과학적 근거와 정확한 팔레트

### 브랜드 컬러 `#c4756e` 심리학적 타당성

- 사용자는 제품과의 첫 상호작용에서 **90초 이내에 판단**을 형성하며, 그 중 **62-90%가 색상에 의해 결정**됨 (Singh, 2006)
- 테라코타는 저채도 따뜻한 톤으로 긴급성/위험 신호 없이 감정적 온기를 활성화
- 따뜻한 색상의 e-commerce 사이트가 구매 의도에 **긍정적 영향** (Wu et al., 2008)
- 한국 오방색(五方色)의 적(남/화)과 황(중앙/토) 사이에 위치하여 문화적으로도 자연스러움

### CTA 버튼 색상: Von Restorff 효과 활용

- 따뜻한 테라코타 지배적 페이지에서 **보색 틸(`#2B7A78`)**을 1차 CTA로 사용하면 Von Restorff 고립 효과(isolation effect)로 시각적 위계가 극대화됨
- 따뜻한 톤 버튼이 차가운 톤의 **2.1배 CTR** 기록 (Hotjar 2026)

### 에이전트 규칙

- 1차 전환 CTA는 반드시 `#2B7A78`(딥 틸) + 백색 텍스트
- 보조 CTA와 브랜드 일관 요소는 `#A85248`(리치 클레이)
- 테라코타 배경 위 텍스트는 `#1F1F1F` 이하 어두운 색만 사용 (대비 7.8:1 이상)
- `#c4756e`는 백색 배경 대비 ~3.43:1로 WCAG AA 본문 텍스트 기준(4.5:1)에 **미달** → 본문에 직접 사용 금지. 대형 헤드라인(≥24px), 장식 요소, 배경에만 사용

## 2. 타이포그래피: Gowun Batang의 과학과 정밀 타입 스케일

### 세리프 모바일 가독성

- 웹사이트 가독성에서 세리프/산세리프 특성이 읽기 속도와 사용자 선호에 차이가 없음 (Vecino et al., 2022)
- 레티나 디스플레이 시대에 두 분류 모두 동등하게 유효 → **Gowun Batang은 모바일에서 가독성 패널티 없이 사용 가능**

### 감성 타이포그래피

- 서체 선택만으로 긍정적 소비자 반응이 **최대 13% 증가** (Monotype × Neurons, 2022)
- 캘리그라피에 가까운 글자가 더 깊은 본능적 감정 반응을 유발
- 동아시아 사용자들은 전통 붓질 느낌을 보존한 서체를 **가장 신뢰**한다고 평가 (Monotype, 2024)
- Gowun Batang의 연필 필기 기반 디자인("고운" = 곱고 섬세한)은 이 연구 결과와 정확히 정렬됨

### 정밀 타입 스케일 (모바일 우선)

```
에이전트용 타입 토큰:

--font-display:    'Gowun Batang', serif     (감성 콘텐츠)
--font-ui:         'Noto Sans KR', sans-serif (기능 UI)

--text-4xl:  2rem     (32px) — Hero 헤드라인    weight: 700  line-height: 1.3
--text-3xl:  1.75rem  (28px) — H1 페이지 제목    weight: 700  line-height: 1.3
--text-2xl:  1.5rem   (24px) — H2 섹션 제목      weight: 700  line-height: 1.35
--text-xl:   1.25rem  (20px) — H3 서브섹션       weight: 700  line-height: 1.4
--text-lg:   1.125rem (18px) — H4 카드 제목      weight: 700  line-height: 1.4
--text-base: 1rem     (16px) — 본문, 입력, CTA 버튼  weight: 400  line-height: 1.75
--text-sm:   0.875rem (14px) — 캡션, 보조        weight: 400  line-height: 1.5
--text-xs:   0.75rem  (12px) — 레이블, 메타데이터   weight: 500  line-height: 1.4
```

### 에이전트 절대 규칙

- 입력 필드 폰트는 반드시 **≥16px** (iOS 자동 확대 방지)
- 한글 본문 `line-height`는 **1.75** (CJK 권장 1.7 + 모바일 여유)
- 헤드라인 `letter-spacing: -0.01em`, 본문 `letter-spacing: 0.01em`
- 한글 본문 줄당 **25-35자** → `max-width: 38rem`
- **Gowun Batang**: 브랜드명, 감성 헤드라인, 본문, 선물 설명, 추천 카드
- **Noto Sans KR**: 내비게이션, 버튼, 레이블, 가격, 숫자, 메타데이터

## 3. 6단계 위저드: 인지과학이 설계한 진행 흐름

### 다단계 폼 vs 단일 폼

- 다단계 폼이 단일 폼 대비 **86% 높은 전환율** (HubSpot)
- 다단계 전환 후 11%→46%(4배 이상)로 증가 (BrokerNotes)
- 초기 단계에 시간을 투자한 사용자가 이탈을 꺼리게 되는 **매몰 비용 효과**
- 6필드 이상 단일 폼의 전환율은 최대 15%까지 하락하지만, 각 단계에 1-2개 질문만 배치하면 해결됨

### 진행 표시기: 부여된 진전 효과(Endowed Progress Effect) 적용

- 10칸 스탬프 카드에 2칸을 미리 찍어준 그룹이 빈 8칸 카드 그룹 대비 **34% vs 19% 완료율** (Nunes & Drèze, 2006)

### 에이전트 구현 규칙

```
진행 표시기 구조:
┌─ 4px 수평 프로그레스 바 (상단 고정) ────────────────────┐
│  [■■■░░░░░░░]  "2 / 6 · 관계깊이"                      │
└────────────────────────────────────────────────────────┘

- 바 시작값: 1단계에서 ~10% 채움 (부여된 진전 효과)
- 채움 색: #c4756e (브랜드 테라코타)
- 잔여 색: #E8DCD8 (연한 웜 그레이)
- 완료 단계: ✓ 체크마크 + 터치 가능 (뒤로 이동)
- 전이 애니메이션: 300ms ease-in-out
```

### 단계별 최적 선택지 수 (Hick's Law + Miller's Law 기반)

| 단계 | 질문 | 권장 선택지 수 | 입력 패턴 |
|------|------|-------------|---------|
| 1 | 관계유형 | 5-7개 | 아이콘+텍스트 카드, 단일 열 |
| 2 | 관계깊이 | 3-4개 | 세그먼트 컨트롤 또는 카드 |
| 3 | 기념일 | 6-8개 | 아이콘+텍스트 카드, 2열 그리드 |
| 4 | 예산 | 4-5개 | 필(pill) 버튼, 단일 열 |
| 5 | 취향/의도 태그 | 8-12개 (복수 선택) | 칩/필, 랩 레이아웃 |
| 6 | AI 추천 Top 3 | 3개 (고정) | 상세 제품 카드 |

### 단계 전이 애니메이션 (Card, Moran & Newell, 1983 기준)

- 시각 인식 시간 평균 **230ms**, 모바일 전이에 **200-300ms** 권장

| 인터랙션 | 지속시간 | 이징 | CSS 속성 |
|---------|---------|------|---------|
| 단계 앞으로 | **280ms** | `cubic-bezier(0.16,1,0.3,1)` ease-out | transform, opacity |
| 단계 뒤로 | **250ms** | ease-out | transform, opacity |
| 옵션 선택 피드백 | **150ms** | ease-out | transform, background |
| 버튼 누름(스케일) | **80ms** 다운 / **200ms** 업 | ease-in / spring | transform |
| 프로그레스 바 채움 | **300ms** | ease-in-out | width |
| 에러 진동 | **300ms** (3회 진동) | ease-in-out | transform |
| AI 결과 카드 스태거 | **350ms** + 100ms 딜레이 | ease-out | transform, opacity |

### 에이전트 절대 규칙

- `transform`과 `opacity`만 애니메이션. `width`, `height`, `top`, `left` 등 레이아웃 속성 애니메이션 금지 (60fps = 16.67ms 프레임 예산 유지)

## 4. 감성 디자인: 선물 심리학과 AI 신뢰 구축

### Norman의 3단계 감성 디자인 프레임워크 적용

- **본능적 수준(Visceral)**: 랜딩 페이지의 아름다움, 따뜻한 팔레트, 우아한 타이포그래피
- **행동적 수준(Behavioral)**: 매끄러운 6단계 흐름과 빠른 AI 응답
- **반성적 수준(Reflective)**: "마음을 새기는" 내러티브와 선물 뒤의 이야기
- 심미적 사용성 효과(aesthetic-usability effect): 아름다운 인터페이스가 실제 사용성도 높게 인식되게 함 (d=0.86, Sonderegger & Sauer, 2010)

### 선물 심리학의 핵심 비대칭

- 증여자는 "매력적이고 인상적인" 선물에, 수혜자는 "실용적이고 쓸 수 있는" 선물에 가치를 둠 (Baskin et al., 2014)
- 즉각적 "와!" 반응에 최적화하면 장기 만족을 소홀히 하는 편향 존재

### 에이전트 UX 문구 규칙

- 추천 카드에 **감동 + 실용** 양면 프레이밍: "열었을 때 기쁨 + 오래 쓸 수 있는 실용성"
- 39%의 사용자가 선물 선택에 **불안**을 느낌 (Mixbook 2025) → 불안 감소 언어 사용: "천천히 함께 찾아봐요"
- "완벽한 선물" 약속 금지 → "마음이 전해지는 선물" 프레이밍

### AI 추천 신뢰 구축 (XAI 연구 기반)

- 설명 가능한 AI(XAI)는 신뢰도를 **M=3.2→4.1**(p<.001)로, 구매율을 **49%→68%**로 끌어올림 (Rahman, 2025)
- 사용자의 72%가 **AI 언어의 톤과 명확성이 신뢰에 직접 영향**을 미친다고 보고 (NNGroup, 2024)

### 에이전트 규칙

- 추천 결과에 반드시 이유 한 줄 포함: "OO님이 좋아하시는 감성적 스타일과 5만원대 예산에 맞춰 골랐어요"
- 매칭 점수 표시: "궁합 92%" + 간단한 근거
- "AI가 추천했어요" 라벨 일관 사용
- "다른 추천 보기" 옵션으로 사용자 통제감 제공
- 절대주의 언어 금지("완벽한!") → "이런 선물은 어떨까요?" 제안형 어조

### 추천 카드 UI 구조 (카드 기반 패턴 연구)

- 카드당 **100자 이내** 요약 권장 (Andrew Coyle, 전 Google)

```
추천 카드 구조 (모바일, 단일 열):
┌──────────────────────────────────┐
│  [라이프스타일 이미지]  60% 높이  │  ← 4:3 고정 비율
│                                  │
│──────────────────────────────────│
│  선물 이름  (18px, Batang Bold)  │
│  ₩49,000  (16px, Noto Sans)     │
│  ★ 궁합 92% · "따뜻함을 전하는"  │
│  "왜 이 선물?" ▸ (접을 수 있음)  │  ← XAI 투명성
│  [♡ 저장]         [선물하기 →]   │  ← 1차 CTA: #2B7A78
└──────────────────────────────────┘

카드 간격: 16px
카드 border-radius: 16px (--radius-xl)
카드 shadow: 0 4px 6px rgba(0,0,0,0.07)
카드 패딩: 16px
정확히 3장만 표시 (Iyengar의 선택 과부하 연구)
```

## 5. 모바일 우선: 한국 디바이스 시장과 터치 최적화

### 한국 디바이스 컨텍스트

- 한국 모바일 시장에서 삼성이 **~80-84%**, 애플이 **~16-18%** (Counterpoint Research, 2025)
- 한국의 스마트폰 보급률은 **95%**, 5G 침투율은 **~97.4%**

### 에이전트 기본 뷰포트: 360px (삼성 Galaxy S/A 시리즈 표준)

```
브레이크포인트 (min-width 미디어 쿼리로 상향 확장):

--bp-mobile-l:  480px
--bp-tablet:    768px
--bp-desktop:   1024px
--bp-desktop-l: 1280px
```

### 터치 타깃: Fitts' Law와 엄지 영역

- MIT Touch Lab: 평균 손가락 패드 **1.6-2cm**, 엄지 **2.5cm**
- 75%의 사용자가 엄지로 조작, 49%가 한 손으로 사용 (Steven Hoober)
- 하단 자연 영역 96%, 중간 61%, 상단 코너 ~25%로 극적 차이

| 요소 | 최소 크기 | 권장 크기 | 근거 |
|------|---------|---------|------|
| **주 CTA 버튼** | 48px 높이 | **52px**, 전체 너비 | Material 48dp + 여유 |
| **옵션 카드** | 48×48px | **56-64px** 높이 | 아이콘+텍스트 수용 |
| **태그 칩** | 36px 높이 | 44px 터치 영역 | 패딩으로 히트 영역 확장 |
| **뒤로 가기** | 44px | 44px | Apple HIG 최소 |
| **타깃 간 간격** | 8px | **12-16px** | 카드 간, 오탭 방지 |
| **화면 좌우 패딩** | — | **16-20px** | 모바일 표준 마진 |

### 에이전트 배치 규칙

- 1차 CTA("다음", "선물하기")는 화면 **하단 40%**(엄지 자연 영역)에 고정
- 뒤로 가기/취소는 상단 좌측(비파괴적)
- 하단 내비게이션 항목 **≤5개**

## 6. 로딩 성능과 스켈레톤 스크린

- 로드 시간 3초에서 **53%가 이탈** (Google "Need for Mobile Speed")
- 각 추가 초마다 전환율이 **4.42% 하락** (HubSpot)
- Amazon은 **100ms 지연당 매출 1% 감소** 보고
- 스켈레톤 스크린은 스피너 대비 체감 속도 **20-30% 향상**, 이탈률 **9-20% 감소**

### 에이전트 AI 대기 화면 규칙

- 제네릭 스피너 금지 → "마음을 읽고 있어요..." 텍스트 + 브랜드 로고 펄스 애니메이션
- 3단계 텍스트 진행: "취향 분석 중 → 선물 매칭 중 → 최적 선물 선별 중"
- 스켈레톤: 최종 카드 레이아웃과 동일한 구조, 뉴트럴 그레이 플레이스홀더
- 쉬머 애니메이션: 좌→우 물결, **~500ms 주기**
- Core Web Vitals 목표: LCP < 2.5s, INP < 200ms, CLS < 0.1

## 7. 접근성: WCAG AA + KWCAG 준수 체크리스트

WCAG 2.1/2.2 AA와 한국 장애인차별금지법(2008) 산하 **KWCAG 2.1**(KS X OT 0003)을 동시에 만족해야 함. 미준수 시 최대 **3천만 원 과태료** 및 형사 처벌 가능성.

### 에이전트 접근성 절대 규칙 [필수 구현]

1. `<html lang="ko">`
2. 모든 폰트 대비: 본문 ≥4.5:1, 대형 텍스트(≥24px) ≥3:1
3. 비텍스트 요소(버튼 테두리, 아이콘) ≥3:1 (WCAG 1.4.11)
4. 입력 폰트 ≥16px (iOS 자동 확대 방지)
5. 320px 너비에서 가로 스크롤 없음 (SC 1.4.10 Reflow)
6. 모든 인터랙티브 요소 키보드 탐색 가능
7. 포커스 표시기: `outline: 2px solid #2B7A78; outline-offset: 2px`
8. 아이콘 전용 버튼에 aria-label 한글 제공: `aria-label="선물 추천 시작"`
9. AI 처리 상태에 `aria-live="polite"` 적용
10. 시맨틱 HTML: `<nav>`, `<main>`, `<header>`, `<footer>`
11. 모든 `<input>`에 명시적 `<label>` 연결
12. `prefers-reduced-motion` 지원:
    ```css
    @media (prefers-reduced-motion: reduce) {
      * { animation-duration: 0.001ms !important;
          transition-duration: 0.001ms !important; }
    }
    ```
13. 200% 텍스트 확대 시 기능 손실 없음
14. 멀티터치 제스처에 단일 포인터 대안 제공

## 9. 한국 문화 맥락과 브랜드 보이스 원칙

- 한국 UX의 핵심 이중성: 네이버/카카오 같은 **고밀도 슈퍼앱**에 익숙하면서도 **스칸디나비안 미니멀리즘**을 선호
- 마로는 "깨끗한 레이아웃 + 따뜻한 감성"의 중간 지점을 점유
- "빨리빨리" 문화를 반영하여 추천 완료까지 **60초 이내** 목표
- 한국인은 감사에 **부채감(indebtedness)**을 더 강하게 결합 → 예산 단계에서 상호성(互惠性)을 고려한 가격 범위를 제시하고, "부담 없는 마음"을 키워드로 사용

### 에이전트 브랜드 보이스 규칙

| 원칙 | 쓰는 표현 | 쓰지 않는 표현 |
|------|---------|-------------|
| 협업적 | "함께 찾아볼까요?" | "추천 결과입니다" |
| 겸손한 확신 | "이런 선물은 어떨까요?" | "완벽한 선물을 찾았습니다!" |
| 감정 인식 | "소중한 분에게 마음을 전하세요" | "상품을 선택하세요" |
| AI 투명성 | "AI가 분석한 결과예요, 마지막 선택은 당신의 마음이에요" | (설명 없는 결과 나열) |
| 불안 감소 | "천천히 골라도 괜찮아요" | "빠르게 결정하세요" |

## 10. 에이전트가 모든 UI 판단에서 따라야 할 10가지 최상위 원칙

1. **색상 대비 우선**: 모든 텍스트 대비 ≥4.5:1. `#c4756e`는 본문 금지, ≥24px 요소만 허용 (WCAG 2.1 SC 1.4.3)
2. **CTA는 보색으로**: 1차 CTA는 `#2B7A78` 딥 틸. 따뜻한 페이지에서 차가운 버튼이 Von Restorff 효과로 최대 주목
3. **한글 행간 1.75**: CJK 텍스트는 라틴보다 50% 넓은 행간 필요
4. **터치 타깃 ≥48px**: Material Design 최소, 1차 CTA는 52px 전체 너비 하단 고정 (Fitts' Law)
5. **단계당 1개 질문**: 각 단계에 3-7개 선택지. 인지 부하 최소화 (Hick's Law, Miller's Law)
6. **부여된 진전 10%**: 1단계에서 프로그레스 바 10% 시작. 완료율 79% 향상 (Nunes & Drèze, 2006)
7. **transform/opacity만 애니메이션**: 280ms ease-out. 레이아웃 속성 절대 금지 (60fps 보장)
8. **AI 이유 필수 표시**: 모든 추천에 한 줄 근거. 신뢰 M=3.2→4.1, 구매율 49→68% (Rahman, 2025)
9. **360px 기준 설계**: 삼성 80%+ 한국 시장. min-width 상향 확장
10. **Gowun Batang = 감성, Noto Sans KR = 기능**: 서체로 감정과 정보를 분리. 캘리그라피 기반 서체가 감정 반응 13% 증가 (Monotype × Neurons, 2022)
